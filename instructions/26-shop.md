# 26 ‚Äî StyxShop

In-game item shop with categories, paginated browsing, and live affordability check. Players spend currency from `IEconomy` and receive items via the world-drop backpack pattern (the only delivery path that survives client PlayerData sync).

## Commands

| Command | What | Perm |
|---|---|---|
| `/shop` or `/s` | Open the shop | `styx.shop.use` |
| `/m ‚Üí Shop` | Open via launcher | `styx.shop.use` |
| `/shop reload` | Reload catalog from disk after editing | `styx.eco.admin` |

## UI flow

**Stage 0 ‚Äî categories** (centered single-column list):
- `[SCROLL]` navigate, `[LMB]` enter category, `[RMB]` close shop
- Shows section name + item count

**Stage 1 ‚Äî items in selected category** (two-column layout):
- Left: paginated list (8 rows visible per page) with item icon + name + price
- Right: large icon, name, quantity badge, description, price, **CAN AFFORD** / **INSUFFICIENT FUNDS** badge
- `[SCROLL]` navigate, `[LMB]` buy, `[RMB]` back to categories

Top-right of the panel shows live balance.

## Two-file config

**`configs/StyxShop.json`** (settings ‚Äî small, framework-watched):
```json
{
  "UsePerm": "styx.shop.use"
}
```

**`data/StyxShop/catalog.json`** (the catalog ‚Äî auto-watched via `/shop reload`):
```json
{
  "Items": [
    {
      "Item": "drinkJarBoiledWater",
      "DisplayName": "",                  // empty = use vanilla item name
      "Description": "Three jars of clean water.",
      "Icon": "",                         // empty = use the item's atlas sprite
      "Count": 3,
      "Quality": 1,
      "Price": 5,
      "Perm": "",                         // empty = visible to all (with UsePerm)
      "Section": "Food"                   // empty = "General"
    },
    ...
  ]
}
```

### Per-entry fields

| Field | Required | Notes |
|---|---|---|
| `Item` | yes | Canonical item name from `items.xml`. Verify via `data/game_refs/items.md` (regenerated by StyxGameRefs). |
| `Count` | yes | Stack size delivered per purchase. |
| `Quality` | yes | Item quality 1‚Äì6 (matters for tools/weapons; basic resources ignore). |
| `Price` | yes | Cost in currency. |
| `Section` | yes-ish | Free-form category. Empty falls back to "General". |
| `DisplayName` | optional | Override the item's localised name. |
| `Description` | optional | Description shown in the right pane. |
| `Icon` | optional | Override sprite name (defaults to `Item` value, which works for any vanilla item). |
| `Perm` | optional | Per-entry gate ‚Äî players without this perm don't see the row. |

## Editing the catalog

After editing `data/StyxShop/catalog.json`:

1. `/shop reload` ‚Äî picks up the catalog into memory immediately. Items work, prices apply.
2. **For NEW items added to the catalog**: their display labels (name, description, icon-key) need to be staged. Do a server restart ‚Äî labels write at shutdown, load at next boot.

So:
- **Tweaking existing item prices / counts**: just `/shop reload`. No restart.
- **Adding new items / new sections**: `/shop reload` THEN restart once for labels.
- **Removing items**: `/shop reload`. The label entries linger until the next-but-one boot but don't visibly hurt.

## Per-entry perm gating

Set `Perm` on an entry to make it donor-only / VIP-only:
```json
{
  "Item": "gunRifleT3SniperRifle",
  "Section": "Weapons",
  "Price": 5000,
  "Perm": "styx.shop.donor",
  ...
}
```

Players without `styx.shop.donor` simply don't see this row. Section item counts adjust automatically per player.

## Failure modes

- **Unknown item name** ‚Äî purchase fails, money is refunded automatically with a chat error. Check exact spelling against `items.xml` (V2.6 item names sometimes diverge from V1.x ‚Äî e.g. `drinkJarBoiledWater` not `drinkJarPureWater`).
- **Insufficient balance** ‚Äî purchase blocked at the chat-affordance check; nothing debited.
- **Economy plugin not loaded** ‚Äî shop reports "Economy not loaded" and refuses to debit.
- **Section with zero visible items for a player** ‚Äî section hidden in their picker. If all sections are zero, shop whispers "No items available to you."

## FX (currency exchange) idiom

The default catalog ships with an `FX` section that sells vanilla **dukes** (`casinoCoin`) for 200 credits each. This lets players convert server currency ‚Üí vanilla currency to spend at vanilla traders. Operator can adjust the exchange rate or add other conversions:
```json
{ "Item": "casinoCoin", "Section": "FX", "Count": 1, "Price": 200, ... }
```

## Where state lives

- `configs/StyxShop.json` ‚Äî small settings file (framework hot-reloaded)
- `data/StyxShop/catalog.json` ‚Äî the catalog (manual `/shop reload`)
- No per-player runtime state ‚Äî purchases are immediate
